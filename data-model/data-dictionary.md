# Data Dictionary — Key Entities

This dictionary covers the 15 most important entities in the system.
For the complete schema, see the [ER diagram](er-diagram.md).

---

## Core Traceability Entities

### `lote_maestro` — Batch Master

The central entity. Every raw cream batch received from a supplier creates one record here.
All downstream operations (lab analysis, production, sales) reference this entity.

| Column | Type | Description |
|--------|------|-------------|
| `id` | UUID PK | Internal identifier |
| `recepcion_id` | UUID FK | Source reception record |
| `folio` | VARCHAR UNIQUE | `CB-YYYYMMDD-PROV-SEQ` — human-readable trace key |
| `estado` | VARCHAR | `Recibido` → `En_Analisis` → `Liberado` / `Rechazado` → `Consumido` / `Vencido` |
| `peso_neto_kg` | DECIMAL | Net weight after tare |
| `fecha_recepcion` | DATE | Date received at plant |
| `fecha_vencimiento` | DATE | Expiry date (enforced at dispatch) |
| `recibido_por` | UUID FK → empleado | Operator who received the batch |

**Constraints:** State machine enforced by CHECK; `folio` generated by DB function.
**Relations:** Referenced by 23+ foreign keys across all modules.

---

### `recepcion_crema` — Raw Cream Reception

Created when a truck arrives with raw cream. Triggers automatic creation of `lote_maestro`.

| Column | Type | Description |
|--------|------|-------------|
| `proveedor_id` | UUID FK | Supplier |
| `fecha` | DATE | Reception date |
| `temperatura_c` | DECIMAL | Temperature at reception (HACCP critical limit) |
| `acidez_dornic` | DECIMAL | Acidity in Dornic degrees (quality indicator) |
| `peso_bruto_kg` | DECIMAL | Gross weight (truck + cream) |
| `tara_kg` | DECIMAL | Tare weight |
| `peso_neto_kg` | DECIMAL GENERATED | `peso_bruto_kg - tara_kg` |

---

### `analisis_laboratorio` — Lab Analysis

LIMS record for each batch. A `Conforme` result signed by `Calidad_Aseguramiento` is
required before a batch can be released for production.

| Column | Type | Description |
|--------|------|-------------|
| `lote_id` | UUID FK | Batch being analyzed |
| `resultado` | VARCHAR | `Conforme` or `No_Conforme` |
| `conforme` | BOOLEAN | Derived flag |
| `observaciones` | TEXT | Notes from the analyst |
| `firmado_por` | UUID FK → empleado | Must have role `Calidad_Aseguramiento` |
| `fecha_firma` | TIMESTAMP | Timestamp of digital signature |

**Governance:** Signing analyst must be a different person from the batch receiver
(segregation of duties — enforced by trigger).

---

### `orden_produccion` — Production Order

Represents a single production run. Links the cream batch (input) to finished product (output).

| Column | Type | Description |
|--------|------|-------------|
| `activo_id` | UUID FK | Equipment used (pasteurizer, separator, etc.) |
| `folio` | VARCHAR UNIQUE | `OP-YYYYMMDD-ACT-SEQ` |
| `estado` | VARCHAR | `Planificada` → `En_Proceso` → `Completada` / `Cancelada` |
| `fecha_inicio` | TIMESTAMP | When production started |
| `fecha_fin` | TIMESTAMP | When production completed |
| `operador_id` | UUID FK → empleado | Responsible operator |

**Governance:** Equipment must have last CIP status = `Limpio_Disponible` (trigger).

---

### `consumos_orden` — Material Consumption

Records which cream batches were consumed by a production order (mass balance input side).

| Column | Type | Description |
|--------|------|-------------|
| `orden_id` | UUID FK | Production order |
| `lote_id` | UUID FK | Cream batch consumed |
| `cantidad_kg` | DECIMAL | Kilograms consumed |

**Governance:** Batch must be `Liberado` with `Conforme` analysis before consumption (trigger).

---

### `resultados_lote` — Production Results

Records what came out of a production order (mass balance output side).
Multiple output types reflect dairy reality (cream → butter + whey + waste).

| Column | Type | Description |
|--------|------|-------------|
| `orden_id` | UUID FK | Production order |
| `tipo` | VARCHAR | `Producto_Principal`, `Co_Producto`, `Merma`, `Reproceso` |
| `cantidad_kg` | DECIMAL | Kilograms produced |
| `folio_pt` | VARCHAR | `PT-YYYYMMDD-ORDEN` — finished product trace key |

## Commercial Entities

### `cliente` — Customer

| Column | Type | Description |
|--------|------|-------------|
| `nombre` | VARCHAR | Legal name |
| `rfc` | VARCHAR UNIQUE | Mexican tax ID (CFDI required) |
| `credito_dias` | INTEGER | Payment terms in days |

---

### `pedido` — Sales Order

| Column | Type | Description |
|--------|------|-------------|
| `cliente_id` | UUID FK | Customer |
| `estado` | VARCHAR | `Pendiente` → `Confirmado` → `Entregado` / `Cancelado` |
| `total_neto` | DECIMAL | Net amount before tax |
| `fecha_pedido` | DATE | Order date |
| `fecha_entrega` | DATE | Agreed delivery date |

---

### `factura` — Invoice

| Column | Type | Description |
|--------|------|-------------|
| `pedido_id` | UUID FK | Source sales order (must be `Entregado`) |
| `folio_fiscal` | VARCHAR | CFDI fiscal UUID |
| `monto_total` | DECIMAL | Total with tax |
| `fecha_timbrado` | DATE | SAT stamp date |

**Governance:** Only created for delivered orders (trigger).

## People and Equipment

### `empleado` — Employee / System User

| Column | Type | Description |
|--------|------|-------------|
| `codigo` | VARCHAR UNIQUE | `EMP-XXX` |
| `nombre` | VARCHAR | Full name |
| `rol` | VARCHAR | One of 5 roles (see below) |
| `pin_hash` | VARCHAR | bcrypt hash — plain PIN never stored |
| `activo` | BOOLEAN | Active status |

**Roles and access:**

| Role | Access |
|------|--------|
| `Operador` | Reception, production, WMS movements |
| `Calidad_Aseguramiento` | Lab analysis, HACCP monitoring, deviation records |
| `Produccion` | Production orders, equipment CIP logs |
| `Ventas` | Sales orders, dispatch |
| `Administrador` | All modules + user management |

---

### `activo` — Production Asset

| Column | Type | Description |
|--------|------|-------------|
| `codigo` | VARCHAR UNIQUE | `ACT-XXX` |
| `tipo` | VARCHAR | `Pasteurizador`, `Separadora`, `Tanque`, etc. |
| `capacidad_kg` | DECIMAL | Maximum batch capacity |
| `estado_cip` | VARCHAR | Current CIP status (derived from CIP log) |

## Financial Entities

### `cuenta_contable` — Chart of Accounts

| Column | Type | Description |
|--------|------|-------------|
| `codigo` | VARCHAR PK | Hierarchical code `X.X.XX` |
| `nombre` | VARCHAR | Account name |
| `tipo` | VARCHAR | `Activo`, `Pasivo`, `Capital`, `Ingreso`, `Gasto`, `Costo` |

---

### `asiento_contable` — Journal Entry

| Column | Type | Description |
|--------|------|-------------|
| `folio` | VARCHAR UNIQUE | `AST-TIPO-REF-FECHA` |
| `estado` | VARCHAR | `Borrador` → `Contabilizado` |
| `periodo_id` | UUID FK | Accounting period (must be open) |
| `concepto` | TEXT | Entry description |

**Governance:** Debit total must equal credit total; period must be open (trigger).

---

### `costeo_lote` — Batch Cost Absorption

Links production costs to a specific batch. Enables cost-per-kg reporting by product.

| Column | Type | Description |
|--------|------|-------------|
| `resultado_id` | UUID FK | Finished product lot |
| `costo_mp_kg` | DECIMAL | Raw material cost per kg |
| `costo_mano_obra_kg` | DECIMAL | Labor cost per kg |
| `costo_cif_kg` | DECIMAL | Overhead cost per kg |
| `costo_total` | DECIMAL GENERATED | Sum of all cost components |
| `costo_por_kg` | DECIMAL GENERATED | `costo_total / cantidad_kg` |
